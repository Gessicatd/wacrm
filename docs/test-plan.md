# Plano de testes

- RLS: uma conta não lê nem grava dados de outra.
- Domínio: criação de projeto, metodologia, etapa e artefato.
- Contratos: estados, tipos de execução, prioridade e schemas JSON.
- Orquestração: retry, falha recuperável, espera por aprovação e retomada.
- Idempotência: mesma execução não duplica artefatos.
- Segurança: agent não altera definições administrativas; viewer não escreve.
- Integração: assessment → projeto → plano/artefato.
